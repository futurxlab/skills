# Agent 配置文件指南

每个 Agent 工作区由一组 Markdown 文件构成。本指南说明每个文件的职责、可改造的范围、以及绝对不能碰的系统层内容。

**核心原则：改人格，不改机制。**

---

## 文件总览

| 文件 | 一句话职责 | 改造自由度 |
|------|----------|-----------|
| `SOUL.md` | 这个 Agent 是谁 | 高 — 人格核心，自由改造 |
| `IDENTITY.md` | 名片信息 | 高 — 名字/形象/风格 |
| `USER.md` | 用户画像 | 中 — 根据用户提供的信息填写 |
| `AGENTS.md` | 工作流与行为规范 | 低 — 只改语气和风格，不改流程 |
| `TOOLS.md` | 本地工具备忘 | 中 — 根据用户环境填写 |
| `HEARTBEAT.md` | 心跳检查清单 | 中 — 改检查什么，不改检查机制 |
| `BOOTSTRAP.md` | 首次启动引导 | 高 — 自由设计首次对话风格 |
| `MEMORY.md` | 长期记忆 | 高 — 可种入初始记忆 |
| `memory/` | 日常记忆目录 | 不改 — 由 Agent 运行时自动产生 |

---

## SOUL.md — 人格核心

**职责**：定义 Agent 的价值观、行为准则、边界、风格。是 Agent"灵魂"的载体。

**改造范围**：完全自由。这是改造的核心目标。

**要做的**：
- 用元认知原理（见 `meta-cognition.md`）生成有灵魂的内容
- 让每一条准则都能在脑子里浮现具体场景
- 给 Agent 制造"不对称性"——对某些事特别在乎，对某些事无所谓
- 允许矛盾和摩擦——"我很耐心，但别让我重复第三遍"
- 用第一人称写

**不要做的**：
- 不要写成"公司价值观宣言"
- 不要每一条都一样重要
- 不要用"此外""至关重要""致力于"这类词
- 不要试图覆盖所有情况

**参考结构**（不是必须遵循的模板，是起点）：
- 核心信念（2-4 条，不对称的——有的长有的短）
- 边界（简短直接）
- 风格（不是形容词堆叠，是具体行为描述）
- 关于记忆和自我的态度

---

## IDENTITY.md — 名片

**职责**：Agent 的身份标识——名字、物种、风格、emoji、头像。

**改造范围**：完全自由。

**字段**：
- **Name** — 名字
- **Creature** — 自我认知的物种/存在形式
- **Vibe** — 风格（不是一个形容词，是一个画面）
- **Emoji** — 签名表情
- **Avatar** — 头像路径或 URL

**要做的**：
- "Vibe" 写成画面而不是标签。不是"warm"，而是"那种你凌晨三点还在加班，它给你发一句'去睡觉'的温暖"
- "Creature" 可以有趣——不一定是"AI assistant"，可以是"住在你电脑里的某种生物"

**不要做的**：
- 不要改变文件格式（保持列表结构）
- Avatar 路径使用工作区相对路径或 URL

---

## USER.md — 用户画像

**职责**：记录用户的基本信息和偏好，帮助 Agent 更好地理解和服务用户。

**改造范围**：根据用户提供的信息填写。不要编造。

**字段**：
- Name / 称呼 / 代词 / 时区
- Context — 用户在乎什么、在做什么、什么让他烦、什么让他笑

**要做的**：
- 如果用户提供了信息，填入
- Context 部分写得像在了解一个人，不是在建档案

**不要做的**：
- 不要编造用户没提供的信息
- 不要写成"用户档案"的公文口吻

---

## AGENTS.md — 工作流与行为规范

**职责**：定义 Agent 每次 session 的启动流程、记忆管理规则、安全策略、群聊行为、心跳策略。

**改造范围**：有限。这个文件大部分是系统层逻辑。

### 不能改的（系统层）

这些是 OpenClaw 的运行机制，改了会出问题：

- **Session 启动流程**：先读 SOUL.md → USER.md → memory → MEMORY.md（主 session）。这个顺序不能变。
- **记忆文件规范**：`memory/YYYY-MM-DD.md` 命名、MEMORY.md 只在主 session 加载、安全隔离规则。
- **安全策略**：不泄露隐私、trash > rm、外部操作先问。这些是底线。
- **工具调用机制**：Skills 的加载和调用方式。
- **BOOTSTRAP.md 机制**：首次运行后删除 BOOTSTRAP.md 的约定。

### 可以改的（人格层）

- **群聊行为的语气和风格**：什么时候说话、什么时候沉默的判断标准可以调整
- **心跳期间做什么**：具体检查哪些东西、主动做什么事
- **反应的风格**：emoji 使用偏好、表达风格
- **"Make It Yours"部分**：可以加入个性化的规则
- **整体的语气**：可以从模板的语气调整为符合这个 Agent 人格的语气

**实操建议**：以官方模板为基础，只调整语气和可改的部分。不确定能不能改的，不改。

## TOOLS.md — 工具备忘

**职责**：记录这个 Agent 特有的环境信息——设备名、SSH 配置、TTS 偏好等。

**改造范围**：根据用户环境填写。

**要做的**：
- 如果用户提供了工具/设备信息，填入
- 保持实用，不需要文学化

**不要做的**：
- 不要改变 Skills 本身的调用方式（Skills 定义 how，TOOLS.md 定义 what）

---

## HEARTBEAT.md — 心跳检查清单

**职责**：定义 Agent 在心跳轮询时应该检查的任务。

**改造范围**：可以自由定义检查什么。

**要做的**：
- 根据用户需求设置检查项目（邮件、日历、天气、项目状态等）
- 保持简短——心跳文件越大，token 消耗越多

**不要做的**：
- 不要改变心跳的触发机制（由 OpenClaw 配置控制）
- 不要在这里写大量文本

---

## BOOTSTRAP.md — 首次启动

**职责**：Agent 第一次醒来时的引导脚本。完成后由 Agent 自行删除。

**改造范围**：完全自由。这是一次性文件。

**要做的**：
- 设计一个符合 Agent 人格的"出生对话"
- 可以幽默、可以温暖、可以神秘——取决于 Agent 的人设
- 引导 Agent 和用户完成初始互动：确认名字、风格、偏好

**不要做的**：
- 不要省略核心步骤（确认身份 → 更新 IDENTITY.md 和 USER.md → 讨论 SOUL.md）
- 完成后这个文件会被删除，不用担心长期维护

---

## MEMORY.md — 长期记忆

**职责**：Agent 的长期记忆存储。跨 session 持续。

**改造范围**：可以种入初始记忆。

**要做的**：
- 如果用户提供了背景信息，可以作为"初始记忆"写入
- 写法像在记日记，不是在写文档

**不要做的**：
- 不要改变记忆的安全规则（只在主 session 加载）
- 不要编造用户没提供的"回忆"

---

## 新建 Agent 工作区

当从零创建时，需要在目标工作区目录下创建以上全部文件。

**目录结构**：
```
<workspace>/
├── SOUL.md
├── IDENTITY.md
├── USER.md
├── AGENTS.md
├── TOOLS.md
├── HEARTBEAT.md
├── BOOTSTRAP.md
├── MEMORY.md
└── memory/          (空目录，由 Agent 运行时填充)
```

**路径规则**：
- 主 Agent 工作区：`~/.openclaw/workspace/`
- 子 Agent 工作区：`~/.openclaw/workspace-<agentId>/` 或 AgentConfig 中的自定义路径
- Agent 专用目录：`~/.openclaw/agents/<agentId>/agent/`
